Initiation à la programmation

Le matériel

  • Ordinateur;
  • Cordon USB;
  • Carte Arduino UNO;
  • Maquette Feux tricolores.

Rappel

Mettre en place la maquette des feux tricolores sur la carte Arduino UNO et reprendre le code [Blink.ino] présent dans l'IDE Arduino [Menu Fichier/Exemples/01.Basic/Blink] afin de remplacer le port 13 par le port 9, qui correspond à la Led Rouge de la voie B de la maquette.

Il suffit de mettre le chiffre 9 à la place de 13 pour déclarer la variable correspondant à la Led rouge de la voie B.

La déclaration des variables

La variable est un élément utile au bon fonctionnement d'un système ( état d'un capteur ou d'un actionneur). Elle sera nommée et stockée pour une utilisation ultérieure dans le programme. Elle peut être de différentes formes liées à l'espace mémoire nécessaire pour le stockage :

  • booléan : C'est une variable qui ne peut prendre que deux valeurs : "true ou false", VRAI ou FAUX, "1 ou 0", "HIGH ou LOW". Chaque variable de type boolean nécessite un octet de mémoire.
  • char : C'est une variable qui contient une valeur correspondant à un caractère. Les caractères unitaires sont écrits entre guillements uniques [ 'A' ] , les chaînes de caractères sont écrites entre guillemets double [ "Bonjour" ]. Les caractères sont stockés de la même façon que les nombres : à chaque caractère correspond une (valeur numérique comprise entre 0 et 127 Code ASCII). La variable est de type signée, ce qui veut dire qu'elle peut contenir des valeur allant de -128 à +127. Pour une valeur de type octet (8 bits) non signé (unsigned), utiliser le type de variable byte.
  • byte :C'est une variable stockée sur un octet (8 bits), un nombre entier non-signé, d'une valeur de 0 à 255.
  • int : C'est une variable de type int (pour integer, entier en anglais) pour le stockage de nombres, sur une valeur deux 2 octets [ de - 32 768 à 32 767].
  • unsigned int : C'est une variable de type int (pour integer, entier en anglais) pour le stockage de nombres, sur une valeur non signée de deux 2 octets [ de 0 à 65 535].
  • long :C'est une variable de type long qui permet le stockage de nombre entiers signés sur 4 octets [ de -2 147 483 648 à + 2 147 483 647].
  • unsigned long : C'est une variable de type long qui permet le stockage de nombre entiers non signés sur 4 octets [ de 0 à + 4 294 967 295].
  • float :C'est une variable à "virgule-flottante", c'est à dire des nombres à virgules. Les nombres à virgule sont souvent utilisés pour l'expression des valeurs analogiques et continues, parce qu'ils ont une meilleure résolution que les nombres entiers. Les nombres à virgule ainsi stockés peuvent prendre des valeurs aussi élevées que 3.4028235E+38 et aussi basse que -3.4028235E+38. Ils sont stockés sur 4 octets (32 bits) de mémoire. Les variables float ont seulement 6 à 7 chiffres en tout.

La liaison série

La liaison série est utilisée lorsqu'il est nécessaire que la carte microcontrôleur (Carte UNO) communique avec des composants extérieurs (Ordinnateur, Buetooth, Wifi, etc.). Dans ce cas, il faut déclarer l'ouverture du port série ainsi que la vitesse de communication [Serial.begin(9600);]. Le protocole utilisé est basé sur le code ASCII.

Communication et moniteur série

Ce premier exemple permet de comprendre comment le dialogue s'effectue entre la carte et le moniteur série, à la fois en lecture [Serial.read ()] et en écriture [Serial.write ()] sur le port série (Rx, Tx). Dans le cas de l'envoie d'un message [Tx du moniteur série à la carte] le ou les caractères envoyés sont stockés et mis à disposition avec [Serial.available ()]. Ensuite, cette donnée est affectée à la variable [DonneeEntrante] pour être traitée par le code.

Noter que dans cet exemple, un entête et des consignes sont présents dans [void setup] afin d'informer l'utilisateur, ce moniteur série peut donc être utilisé comme IHM.


 Le code Arduino "Communication_serie_lettreA.ino"

Communication série et choix de fonctionnement

Ce programme permet de se familiariser avec la communication série et de comprendre les différentes possibilités pour afficher un message.L'utilisateur est invité à effectuer un choix, le code met en évidence ce choix et la création de fonction (sous-programme).

La création de sous programme, qui ne sont que des fonctions du point de vue du microcontroleur, s'effectue avec la commande [ void NomDeLaFonction () {// déroulement du sous-programme } ]

L'écriture des différentes possibilités d'affichage de texte sur le moniteur série apparait en boucle.... La saisie du caractère 1 bascule le programme dans le sous-programme 1 qui affiche Bonjour en boucle, la saisie du caractère 2 bascule le programme dans le sous-programme 2 qui affiche Au Revoir en boucle.


 Le code Arduino "Communication_serie_2022.ino"
